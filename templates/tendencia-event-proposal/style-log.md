# Style Log

Track design decisions and reusable patterns across components.

## Architecture
- **Composition model**: React-like JS fragment loader. Each slide lives in `components/<name>.html`
  as a **fragment** — only the `.slide` container (1920×1080) plus an optional `<style>` block for
  component-specific CSS (injected `<style>` still applies to the shared DOM). No `<html>/<head>/<body>`.
- **Deck shell**: `presentation.html` holds Tailwind CDN + `tokens.css`, lists one
  `<div class="slide-mount" data-slide="NAME">` per slide, and fetches/injects each fragment.
  Add a slide = drop a file in `components/` + add one mount line.
- **Serving**: `fetch()` is blocked on `file://` — run `python -m http.server` and open via
  `http://localhost:8000/presentation.html`. (This is the one deviation from "open directly";
  chosen for single-DOM composition + clean PDF export.)
- **Asset paths**: fragments reference `assets/...` relative to `presentation.html` at the root.
- **Print**: `@page { size: 1920px 1080px }` + `break-after: page` per slide-mount for PDF export.

## Component 5: practical-info
- **Layout**: **CREATIVE DEVIATION (user-requested)** — the source used a central vertical timeline with
  left/right branches; replaced with a **3×2 card grid**. Content preserved 1:1, presentation reimagined.
- **ICON SET (new project-wide pattern)**: Tabler Icons via **webfont CDN**
  (`@tabler/icons-webfont@3.31.0`), used as `<i class="ti ti-xxx"></i>`. Webfont (not JS) chosen because it
  works inside injected fragments with no `createIcons()` timing issue. The `<link>` lives in the fragment
  and loads fine via innerHTML injection (verified). Use this approach for any future icons.
- **Card spec (reusable, "folded-corner / dog-ear")**: matches a reference infographic card, numberless.
  Each card = `.card-wrap` (sets `--accent`) → `.info-card` (`overflow:hidden`, radius 26). Elements:
  • faint tinted body `color-mix(in srgb, var(--accent) 7%, #fff)`, border `… 28%, #fff`;
  • `.dogear` — bold accent triangle at top-left (`clip-path: polygon(0 0,100% 0,0 100%)`), `::after` adds a
    subtle dark crease; clipped to the rounded corner by `overflow:hidden`;
  • `.icon-circle` LEFT (120px, tint `… 20%, #fff` bg, accent glyph 56px);
  • `.card-text` (heading accent uppercase 24px bold; value 26px `--color-text-primary`);
  • `.card-dots` "•••" bottom-right (`… 50%, #fff`).
  One `--accent` var per card drives body tint, dog-ear, icon tint, heading, and dots. `color-mix()` verified
  in headless Chrome. (Earlier rotated "back-sheet" version was too subtle — replaced by the dog-ear.)
- **Per-item colors** (from the source's connector-line colors): gold `--brand-accent`, royal `--brand-blue`,
  gray `--color-text-secondary`, green `--brand-secondary`, orange `--brand-accent-dark`, teal `--brand-primary`.

  NOTE: Tabler webfont needs a beat to load before the stars paint — use `--virtual-time-budget` when
  screenshotting (short sleeps showed empty star slots).
- **Hotel row**: `.hotel-row` flex (num gold + name dark, stars pushed right via `margin-left:auto`) with a
  `--color-border` bottom divider → reads as a list ready for more entries.
